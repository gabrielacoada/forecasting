```

```
Pipeline: Read problem set → Build context brief from lectures → Fetch data → Analyze → Visualize → Assemble notebook. The context brief (Step 3) uses the course-knowledge agent to pull professor's explanations from lecture materials so discussions match her terminology and expectations.

## Important Rules
- All code is Python
- Jupyter notebooks are the deliverable for problem sets
- Discussion sections should use Professor Pesavento's terminology from lectures
- Research facts must be attributed to sources — never fabricate
- The climate project analysis must stay industry-level, not bank-specific
- Models should be transparent and explainable — no black-box ML

## Validation Rules (ALWAYS FOLLOW)
- **After generating any notebook, ALWAYS run it and check that outputs are non-empty.** Do not assume code works because it has no syntax errors.
- **After creating any plot, verify it contains actual data.** Check that arrays/DataFrames being plotted are not empty. If a plot has no visible data, STOP and report the issue before continuing.
- **After fetching or loading any data, print the shape and first few rows.** If a DataFrame is empty or has unexpected dimensions, STOP and report it.
- **After any data merge, join, or filter, check that rows weren't dropped unexpectedly.** Print before/after row counts.
- **If any variable comes back as NOT AVAILABLE, NaN, or empty, flag it immediately** — do not silently proceed and leave blank outputs.
- **Before moving to the next pipeline step, verify the current step produced valid output.** Each step's output is the next step's input — garbage in, garbage out.
- When in doubt: print it, plot it, check it. Never assume.
